# ТЕХНИЧЕСКОЕ ОПИСАНИЕ ПРОГРАММЫ
# "Визуализатор штампов электронной подписи"
# Версия 1.0

## 1. Назначение программы
Программа предназначена для визуализации информации об электронных подписях в PDF-документах путем добавления графических штампов и протоколов проверки.

## 2. Основные функции
- Визуализация информации из файлов электронной подписи (.sig)
- Добавление графических штампов в PDF-документы
- Создание протоколов проверки подписей
- Обработка доверенностей в формате XML
- Визуальное размещение элементов на документах

## 3. Технические характеристики
- Язык программирования: Java 21
- Графический интерфейс: JavaFX
- Библиотеки работы с PDF: Apache PDFBox 2.0.29
- Формат входных данных: PDF, SIG, XML
- Формат выходных данных: PDF

## 4. Отсутствие функций криптозащиты
Программа НЕ выполняет:
- Проверку криптографической целостности подписей
- Верификацию сертификатов
- Шифрование/дешифрование данных
- Создание электронных подписей
- Криптографические операции любого вида

# РУКОВОДСТВО ПОЛЬЗОВАТЕЛЯ

## 1. Установка и запуск
### Требования к системе:
- Java Runtime Environment 21 или выше
- 2 ГБ оперативной памяти
- 100 МБ свободного места на диске

### Запуск приложения:
java -jar nbdsig-1.0-SNAPSHOT.jar

## 2. Основные операции

### 2.1. Выбор файлов
- Нажмите "Выбрать файлы"
- Выберите PDF документ и файлы подписей (.sig)
- Программа автоматически распознает информацию о подписантах

### 2.2. Настройка типа документа
- Выберите тип документа из выпадающего списка
- Доступные типы: кредитные договоры, банковские гарантии и др.

### 2.3. Добавление доверенности
- Нажмите "Добавить доверенность"
- Выберите XML файл доверенности
- Программа автоматически извлечет данные

### 2.4. Размещение штампов
- Нажмите "Добавить подписи"
- Укажите страницы для размещения штампов
- Программа добавит визуальные элементы

### 2.5. Создание протокола
- Нажмите "Сформировать протокол"
- Выберите место размещения на документе
- Заполните данные протокола

## 3. Выходные данные
Программа создает модифицированные PDF-файлы с добавленными графическими элементами, содержащими информацию о подписях.

# ОПИСАНИЕ АЛГОРИТМОВ

## 1. Алгоритм обработки подписей

Вход: Файлы подписей (.sig)
Выход: Структурированная информация о подписантах

Шаги:
Чтение метаданных из файлов .sig
Парсинг информации о владельце подписи
Классификация подписей по категориям:
Подписанты со стороны банка
Подписанты со стороны клиента
Дополнительные подписанты
Формирование графических представлений


## 2. Алгоритм размещения штампов

Вход: PDF документ, информация о подписях, номера страниц
Выход: Модифицированный PDF документ

Шаги:
Анализ структуры PDF документа
Определение областей для размещения штампов
Генерация графических элементов
Добавление элементов на указанные страницы
Сохранение модифицированного документа


## 3. Алгоритм обработки доверенностей

Вход: XML файл доверенности
Выход: Структурированные данные доверенности

Шаги:
Парсинг XML структуры
Извлечение основных атрибутов:
Номер доверенности
Даты действия
ФИО доверенного лица
Валидация формата данных
Интеграция в визуальное представление

# ОПИСАНИЕ ФОРМАТОВ ДАННЫХ

## 1. Входные форматы

### 1.1. PDF документы
- Стандарт: PDF 1.7 и выше
- Кодировка: Base64 не требуется
- Ограничения: без паролей защиты

### 1.2. Файлы подписей (.sig)
- Формат: Производные от PKCS#7
- Содержимое: Метаданные подписи
- Извлекаемая информация:
    - ФИО владельца
    - Организация
    - Должность
    - Сертифицирующий центр

### 1.3. XML доверенности
- Стандарт: XML 1.0
- Кодировка: UTF-8
- Обязательные элементы:
    - СвДов (НомДовер, ДатаВыдДовер, СрокДейст)
    - СвУпПред (ФИО с атрибутами)

## 2. Выходные форматы

### 2.1. Модифицированные PDF
- Добавленные графические элементы
- Текстовые блоки с информацией
- Сохранение исходной структуры документа

### 2.3. Визуальные элементы
- Графические штампы с текстом
- Таблицы с информацией о подписях
- Протоколы проверки

# МЕРЫ ЗАЩИТЫ ИНФОРМАЦИИ

## 1. Обработка конфиденциальных данных
Программа обрабатывает, но НЕ хранит:
- Персональные данные подписантов
- Реквизиты документов
- Информацию о доверенностях

## 2. Отсутствие криптографических функций
Подтверждается, что программа:
- Не использует алгоритмы шифрования
- Не выполняет проверку ЭП
- Не имеет доступа к закрытым ключам
- Не модифицирует сами подписи

## 3. Временное хранение данных
Все обрабатываемые данные:
- Хранятся только в оперативной памяти
- Удаляются после закрытия приложения
- Не записываются в постоянное хранилище

## 4. Соответствие требованиям
Программа соответствует требованиям для ПО:
- Не содержащего средств криптографической защиты
- Обрабатывающего открытые данные
- Визуализирующего информацию

# ЗАЯВЛЕНИЕ О СООТВЕТСТВИИ

Настоящим подтверждаем, что программное обеспечение
